# Templates

Use these templates as compact guides. Prefer repository-specific templates from `README.md` when present.

## Cleaned-note review draft

```markdown
# Cleaned Raw Notes Review: YYYY-MM-DD

## Work progress

- ...

## Communication and context

- ...

## Decisions / reasoning

- ...

## Problems / blockers

- ...

## Next steps

- ...

## Non-work / reflection / life notes

- ...

## Uncertain corrections

- `raw phrase` -> `possible correction`: reason / uncertainty

## Consistency corrections

- `raw phrase` -> `canonical name`: matched from prior notes / glossary / repeated context
```

Rules:

- Keep this close to the source.
- Do not turn it into a polished report yet.
- Include uncertain corrections only when relevant.
- Use consistency corrections for likely name/project/company/tool recognition errors.
- Ask the user to approve or edit this draft before writing final outputs.

```markdown
- ...

- ...

- ...

## Tomorrow / next steps

- ...

## Review-worthy material

- ...

- State: ...

```

Use `State and advice` only when it helps the user reflect or improve. Keep it direct and evidence-based.
